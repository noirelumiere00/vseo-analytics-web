## 1. インフラ・環境構築

### 1-1. 必須環境変数の設定

| 変数名 | 用途 | 優先度 |
|--------|------|--------|
| `DATABASE_URL` | MySQL接続文字列 | **必須** |
| `JWT_SECRET` | セッション署名用シークレット | **必須** |
| `AWS_ACCESS_KEY_ID` | AWS認証 | **必須**（分析機能に必要） |
| `AWS_SECRET_ACCESS_KEY` | AWS認証 | **必須**（分析機能に必要） |
| `AWS_REGION` | AWSリージョン（デフォルト: us-west-2） | 任意 |
| `BEDROCK_MODEL_ID` | LLMモデル（デフォルト: claude-haiku-4-5） | 任意 |
| `APP_URL` | アプリのURL（CORS・リダイレクトで使用） | **必須** |
| `ADMIN_EMAIL` | 管理者メールアドレス | 推奨 |

**総合: 7.7/10 — MVPとして本番投入可能な水準**

### 8-2. 実装済みの差別化要素

**1. 3ブラウザ合意アルゴリズム（独自性: 高）**
- 3つの独立したシークレットブラウザで同一キーワード検索
- 2/3以上に出現する動画を「アルゴリズム耐性のある勝者」として特定
- パーソナライズバイアスを排除した客観的分析

**2. LLMによる4層分析（深度: 高）**
- 感情分析 + キーフック抽出（動画ごと）
- 勝ちパターン共通性分析（全3ブラウザ出現動画）
- 負けパターン分析（1ブラウザのみ出現動画）
- アスペクト分析（ビジネス視点の多面的評価）

**3. 感情ワード2Dマッピング**
- Russell の感情円環モデルに基づく valence × arousal 座標
- 単純な Positive/Negative を超えた感情の強度・方向性を可視化

**4. キャンペーン効果測定（独自性: 中〜高）**
- Before/After スナップショット比較
- 競合ポジション追跡
- 「波及効果」検出 — 他クリエイターが自分のスタイルを模倣したかを追跡

4. **日本市場特化** — 日本語プロンプト最適化、日本市場のTikTokトレンド理解
