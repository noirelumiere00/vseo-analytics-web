1. Product Overview

VSEO Analytics is a B2B SaaS platform that provides comprehensive short-form video content analysis, primarily focused on TikTok. The platform helps PR agencies, marketing teams, and brand managers understand video content performance through AI-powered sentiment analysis, engagement metrics, and keyword trend analysis.

Core Capabilities (Existing)

FeatureDescription
Triple Search AnalysisSearches the same keyword across 3 independent browser sessions to eliminate algorithmic bias and identify genuinely trending videos
AI Sentiment AnalysisClassifies videos as Positive / Neutral / Negative using LLM analysis of OCR text, audio transcriptions, and visual elements
Engagement MetricsCollects views, likes, comments, shares, and saves for each video
Keyword ExtractionIdentifies trending keywords and hashtag strategies from video content
Cross-Analysis ReportsGenerates comprehensive reports with emotion word maps, aspect-based sentiment, and actionable insights
Trend TrackingMonitors keyword performance over time with historical comparison

Proposed Enhancement: Google Ads Integration

We plan to integrate Google Ads API to bridge the gap between organic video content trends and paid search advertising strategy. This will enable our clients to:

  • Compare trending video keywords with Google Ads search volume and competition data
  • View Google Ads campaign performance alongside video content analysis in a unified dashboard
  • Generate cross-platform reports that correlate video content trends with paid search performance

2.3 Use Case Example

Scenario: A cosmetics brand wants to understand how TikTok video trends for "skincare routine" correlate with their Google Ads search campaign performance.

Workflow:
  1. User runs a VSEO Analysis for keyword "skincare routine" on TikTok
  2. System collects 30+ videos, analyzes sentiment, extracts trending sub-keywords
  3. User links their Google Ads account via OAuth consent
  4. System fetches Google Ads campaign data for related keywords
  5. Dashboard displays: video sentiment trends vs. search ad CTR, trending video keywords vs. Google Ads keyword competition
  6. User discovers high-engagement video keywords that have low competition in Google Ads, identifying new ad opportunities

4. Google Ads API Integration Design

4.1 Integration Scope

Our Google Ads API integration focuses on three primary capabilities:

CapabilityAPI ServicePurpose
Keyword Planning KeywordPlanIdeaService Generate keyword ideas and search volume estimates based on trending video keywords discovered through our analysis
Reporting GoogleAdsService.SearchStream Fetch campaign, ad group, and keyword performance metrics to display alongside video content analysis in our dashboard
Campaign Management GoogleAdsService.Search Read campaign structure and settings to provide context for cross-platform analysis (read-only operations)

4.2 API Resources Used

ResourceOperationsFrequency
customersGET (list accessible accounts)On OAuth connection
campaignsGET (list, metrics)On dashboard load, report generation
ad_groupsGET (list, metrics)On report generation
ad_group_criterions (keywords)GET (list, metrics)On report generation
keyword_plan_ideasPOST (generate ideas)On keyword analysis request
search_term_viewsGET (search terms report)On report generation
Important: Our tool performs read-only operations on customer accounts. We do not create, modify, or delete any campaigns, ad groups, or ads through the API. The only write operation is KeywordPlanIdeaService which generates keyword suggestions without modifying the account.

5.3 Token Management

  • Refresh tokens are encrypted (AES-256) and stored in our MySQL database
  • Access tokens are kept in server memory only, never persisted
  • Tokens are automatically refreshed before expiration
  • Users can revoke access at any time through the VSEO Analytics settings page
  • On revocation, all stored tokens and cached Ads data are permanently deleted

6. API Endpoints & Usage Details

6.1 Keyword Planning Integration

When users run a video analysis, our system extracts trending keywords. Users can then click "Check Google Ads Potential" to see search volume and competition data.

GAQL Query Example:

// KeywordPlanIdeaService.GenerateKeywordIdeas { customer_id: "1234567890", language: "languageConstants/1005", // Japanese geo_target_constants: ["geoTargetConstants/2392"], // Japan keyword_seed: { keywords: ["skincare routine", "スキンケア"] // From video analysis }, keyword_plan_network: "GOOGLE_SEARCH" }

6.2 Campaign Performance Reporting

Users who link their Google Ads accounts can view campaign metrics alongside video analysis results.

GAQL Query Example:

SELECT campaign.id, campaign.name, campaign.status, campaign.advertising_channel_type, metrics.impressions, metrics.clicks, metrics.cost_micros, metrics.conversions, metrics.average_cpc, metrics.ctr, segments.date FROM campaign WHERE segments.date DURING LAST_30_DAYS AND campaign.status = 'ENABLED' ORDER BY metrics.impressions DESC

6.3 Keyword Performance Reporting

GAQL Query Example:

SELECT ad_group_criterion.keyword.text, ad_group_criterion.keyword.match_type, metrics.impressions, metrics.clicks, metrics.ctr, metrics.average_cpc, metrics.cost_micros, metrics.conversions FROM keyword_view WHERE segments.date DURING LAST_30_DAYS AND ad_group_criterion.status = 'ENABLED' ORDER BY metrics.impressions DESC LIMIT 100

6.4 Search Terms Report

GAQL Query Example:

SELECT search_term_view.search_term, metrics.impressions, metrics.clicks, metrics.ctr, metrics.conversions FROM search_term_view WHERE segments.date DURING LAST_30_DAYS ORDER BY metrics.impressions DESC LIMIT 200

8. Rate Limiting & Error Handling

8.1 Rate Limiting Strategy

StrategyImplementation
Request throttlingMax 1 request/second per customer ID using a token bucket algorithm
CachingCampaign and keyword reports are cached for 1 hour in server memory to avoid redundant calls
BatchingMultiple keyword ideas are fetched in a single API call rather than individual calls
Daily limitsSoft limit of 10 keyword planning requests per user per day

8.2 Error Handling

Error TypeHandling
RATE_LIMIT_EXCEEDEDExponential backoff (1s, 2s, 4s, max 3 retries), then queue for later
AUTHENTICATION_ERRORAttempt token refresh; if fails, prompt user to re-authenticate
AUTHORIZATION_ERRORNotify user that account access has been revoked, remove stored tokens
INTERNAL_ERRORRetry once after 5 seconds; log error for investigation
QUOTA_EXCEEDEDDisplay user-friendly message, defer requests to next day

8.3 Logging & Monitoring

  • All API calls are logged with timestamp, customer_id, method, and response status
  • Error rates are monitored; alerts are triggered if error rate exceeds 5% in a 15-minute window
  • API usage metrics are tracked in our admin dashboard (accessible at /admin/logs)

10. Data Security & Privacy

10.1 Data Storage

Data TypeStorageEncryptionRetention
OAuth refresh tokensMySQL (encrypted column)AES-256-GCMUntil user revokes access
Access tokensServer memory onlyN/A (not persisted)Session duration only
Google Ads reports (cached)Server memoryN/A1 hour TTL cache
Customer IDsMySQLAES-256-GCMUntil user revokes access

10.2 Access Controls

  • All API endpoints require authenticated user sessions (cookie-based authentication)
  • Users can only access their own linked Google Ads accounts
  • Admin role has access to system logs but not to user Google Ads data
  • Google Ads API credentials (developer token, client ID, client secret) are stored as environment variables, never in source code

10.3 Data Deletion

  • When a user disconnects their Google Ads account, all related tokens and cached data are immediately and permanently deleted
  • When a user deletes their VSEO Analytics account, all associated data (including Google Ads tokens) is permanently removed
  • No Google Ads data is shared with third parties or used for purposes other than displaying reports to the authenticated account owner

10.4 Infrastructure Security

  • All communication uses HTTPS/TLS 1.3
  • Database connections use SSL
  • Server runs in an isolated container environment
  • Regular security updates applied to all dependencies
